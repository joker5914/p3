# Dismissal Firestore — Collection Schema & Setup Guide

This document is the authoritative reference for the Firestore collections used by Dismissal. **Only three collections are used.** The old `imported_data` collection should be deleted.

---

## Collections Overview

| Collection | Purpose | Written by | Read by |
|---|---|---|---|
| `plates` | Legacy plate registry (CSV import) | Backend import / add_plate.py | Backend on scan (fallback) |
| `plate_scans` | Scan event log | Backend `/api/v1/scan` | Backend dashboard & history |
| `school_admins` | Staff/admin user roles | Backend `/api/v1/users/invite` | Backend auth middleware |
| `guardians` | Guardian/parent profiles | Auto-created on first login | Benefactor endpoints, scan |
| `students` | Student records (linked to guardians) | Benefactor portal | Scan (new vehicle model) |
| `vehicles` | Vehicle records (linked to guardians) | Benefactor portal | Scan (new vehicle model) |
| `schools` | School configuration + enrollment codes | Super admin | School code lookup |

---

## `plates` — Plate Registry

**Document ID:** HMAC-SHA256 of the plate number (keyed with `SECRET_KEY`). Never the raw plate number.

```
plates/{plate_token}
  plate_number_encrypted    : string              Fernet-encrypted plate number
  student_names_encrypted   : string | string[]   Fernet-encrypted student name(s)
  parent                    : string              Fernet-encrypted primary guardian name
  guardian_id_encrypted     : string              Fernet-encrypted guardian email
  school_id                 : string              Plain — used for access scoping
  vehicle_make              : string | null
  vehicle_model             : string | null
  vehicle_color             : string | null
  guardian_photo_url        : string | null        Firebase Storage URL
  student_photo_urls        : string[] | null      Firebase Storage URLs
  authorized_guardians      : array | null         Additional authorized pickup people
    [].name_encrypted       : string               Fernet-encrypted guardian name
    [].photo_url            : string | null         Firebase Storage URL
  imported_at               : string (ISO 8601)
```

**Written by:** `POST /api/v1/admin/import-plates` and `add_plate.py` CLI.

**Read by:** `POST /api/v1/scan` — looks up the plate token to get student/guardian names.

**Delete:** `DELETE /api/v1/plates/{plate_token}` — school_admin only.

## `plate_scans` — Scan Event Log

**Document ID:** Auto-generated by Firestore.

```
plate_scans/{auto_id}
  plate_token             : string              Same token as the plates doc ID
  student_names_encrypted : string | string[]   Fernet-encrypted (copied from plates)
  parent_name_encrypted   : string              Fernet-encrypted (copied from plates)
  timestamp               : timestamp           Local device time (timezone-aware)
  location                : string              Scanner location label
  confidence_score        : number (0.0–1.0)    OCR confidence from RPi scanner
  hash                    : string              HMAC-SHA256 integrity hash
  school_id               : string              Access scoping
```

**Written by:** `POST /api/v1/scan` — every plate detection from the RPi scanner.

**Read by:**
- `GET /api/v1/dashboard` — today's queue (ascending timestamp)
- `GET /api/v1/history` — paginated history with date/name filters
- `GET /api/v1/reports/summary` — statistics

**Delete:** `DELETE /api/v1/scans/clear` — clears all scans for the school (school_admin only).

### Required Firestore Indexes

These composite indexes are defined in `firestore.indexes.json` and deployed automatically via `firebase deploy --only firestore:indexes`:

| Collection | Fields | Order | Used by |
|---|---|---|---|
| `plate_scans` | `school_id` ASC, `timestamp` ASC | Dashboard query |
| `plate_scans` | `school_id` ASC, `timestamp` DESC | History query |
| `plates` | `school_id` ASC, `imported_at` DESC | Plates list |
| `school_admins` | `school_id` ASC, `display_name` ASC | Users list |

## `school_admins` — User Management

**Document ID:** Firebase Auth UID.

```
school_admins/{uid}
  uid             : string              Firebase Auth UID
  email           : string              Plain — not sensitive, used for display
  display_name    : string
  school_id       : string              Access scoping
  role            : "school_admin" | "staff"
  status          : "pending" | "active" | "disabled"
  invited_by_uid  : string              UID of the admin who created this account
  invited_at      : timestamp
  created_at      : timestamp
```

**Written by:**
- `POST /api/v1/users/invite` — creates the record with `status: pending`
- `GET /api/v1/me` — transitions `pending` → `active` on first login
- `PATCH /api/v1/users/{uid}/role` — updates role field
- `PATCH /api/v1/users/{uid}/status` — updates status field
- `Backend/firestore_setup.py` — one-time bootstrap for the first admin

**Read by:**
- `verify_firebase_token()` — checks role & status on every authenticated request
- `GET /api/v1/users` — lists all users for the school
- `GET /api/v1/me` — returns the calling user's profile

**Role definitions:**

| Role | Can do |
|---|---|
| `school_admin` | Everything — import plates, invite users, clear scans, delete plates |
| `staff` | View dashboard, view history, dismiss individual queue entries |

## `students` — Student Records

**Document ID:** Auto-generated by Firestore.

```
students/{auto_id}
  first_name_encrypted : string         Fernet-encrypted
  last_name_encrypted  : string         Fernet-encrypted
  school_id            : string         Links to schools/{id}
  school_name          : string         Denormalized school name

  photo_url            : string | null
  guardian_uid         : string         Firebase UID of parent
  created_at           : string (ISO 8601)
```

**Written by:** `POST /api/v1/benefactor/children`

**Read by:** `POST /api/v1/scan` (new vehicle model), benefactor endpoints

## `vehicles` — Vehicle Records

**Document ID:** Auto-generated by Firestore.

```
vehicles/{auto_id}
  plate_number_encrypted : string       Fernet-encrypted
  plate_token            : string       HMAC-SHA256 of plate (for scanner lookup)
  make                   : string | null
  model                  : string | null
  color                  : string | null
  year                   : string | null
  photo_url              : string | null
  guardian_uid           : string        Firebase UID of owner
  school_ids             : string[]      Campuses where registered (auto-derived)
  student_ids            : string[]      Students authorized for pickup
  created_at             : string (ISO 8601)
```

**Written by:** `POST /api/v1/benefactor/vehicles`

**Read by:** `POST /api/v1/scan` — queries by `plate_token` for vehicle lookup

**Scan flow:** Scanner → tokenize plate → query `vehicles` by `plate_token` → if found, use new model; else fall back to legacy `plates` collection.

## What to delete: `imported_data`

The `imported_data` collection was created by the old `/api/v1/admin/import` endpoint which has been removed. It stored raw/incorrectly-encrypted plate data and is no longer read or written by any part of the system.

**Delete it:** Firebase Console → Firestore → `imported_data` → ⋮ → **Delete collection**.

## First-time setup checklist

```
[ ] 1. Delete the imported_data collection in Firebase Console
[ ] 2. Deploy security rules:  firebase deploy --only firestore:rules
[ ] 3. Deploy indexes:         firebase deploy --only firestore:indexes
[ ] 4. Bootstrap your admin account:
        python Backend/firestore_setup.py \
          --uid  <your Firebase UID> \
          --email you@example.com \
          --name  "Your Name" \
          --school your_school_id
[ ] 5. Log out and back into the admin portal
[ ] 6. Import plates via the Data Importer or add_plate.py CLI
[ ] 7. Verify the plates collection has documents with plate_token as the doc ID
```

## `firmware_releases` — OTA Release Ledger (issue #104)

**Document ID:** semantic version, e.g. `1.2.3`.

```
firmware_releases/{version}
  version              : string                "1.2.3"
  channel              : string                "stable" | "beta" | "hotfix"
  status               : string                "draft" | "published" | "halted" | "archived"
  artifact_storage_path: string                "firmware/releases/1.2.3/dismissal-1.2.3.tar.gz"
  artifact_sha256      : string                hex digest, 64 chars
  signature_ed25519    : string                base64 of the Ed25519 signature
  size_bytes           : number
  signed_at            : string  (ISO 8601)
  signed_by            : string                release engineer identifier
  notes                : string                changelog / release notes (markdown)
  min_compatible_version: string               oldest currently-installed version that may upgrade
  rollout              : object
    stages             : array<{name, percent, min_soak_hours, started_at, completed_at}>
    current_stage      : number                index into stages[] (0 == canary)
    auto_advance       : bool                  if true, advance once min_soak passes with no failures
    halted             : bool                  kill-switch — pauses every Pi
    halt_reason        : string
    failure_threshold  : number                max device failures before auto-halt
  scope                : object
    include_districts  : array<string>          empty == all
    include_schools    : array<string>          empty == all
    exclude_devices    : array<string>          hostname blocklist
  apply_window_local   : { start_hour, end_hour }   evaluated in school's timezone
  metrics              : object
    targeted_count, downloaded_count, applied_count, failed_count, rolled_back_count
  created_at, published_at : string (ISO 8601)
```

**Written by:** `POST /api/v1/admin/firmware/releases` (super_admin) — verifies the manifest's Ed25519 signature against the canonical public key before allowing the doc to land.

**Read by:** `GET /api/v1/scanner/firmware-check` (scanner-role) and the Admin Portal's Firmware page (super_admin / district_admin).

## `device_firmware` — Per-device OTA State (issue #104)

**Document ID:** matches `devices/{hostname}` so admin views can join with one read.

```
device_firmware/{hostname}
  hostname             : string
  current_version      : string                what the Pi is running right now
  previous_version     : string                rollback target — populated on every successful swap
  target_version       : string                what the rollout assigned (may equal current)
  pinned_version       : string | null          super_admin override; takes priority over rollout
  state                : string                "idle" | "assigned" | "downloading" | "verified" | "staged" | "applying" | "health_check" | "committed" | "rolled_back" | "failed"
  state_updated_at     : string  (ISO 8601)
  last_check_at        : string  (ISO 8601)
  last_attempt_at      : string  (ISO 8601)
  last_attempt_version : string
  last_error           : string                short reason on rollback / failure
  rollout_stage        : string                stage name when assigned ("canary" / "early" / ...)
  rollout_assigned_at  : string  (ISO 8601)
  attempts_history     : array                last 10 attempts (rolling window)
    [].at, [].state, [].version, [].error
  created_at           : string  (ISO 8601)
```

**Written by:** Cloud Functions only (Admin SDK).  The OTA agent on the Pi posts state via `POST /api/v1/scanner/firmware-status`; the backend writes here.

**Read by:** Admin Portal's Devices page (firmware column) and the firmware rollout dashboards.

## `platform_settings` — Backend-managed config

```
platform_settings/firmware
  public_key_b64       : string                base64 of the raw 32-byte Ed25519 public key
  rotated_at           : string  (ISO 8601)
  rotated_by           : string                super_admin uid
```

The canonical OTA public key.  Distribution to Pis happens via image provisioning (`/opt/dismissal/keys/firmware.pub`); this Firestore copy is what the backend uses to verify uploaded manifests.  Rotation requires re-imaging the fleet and bumping this doc.

## How the security rules work

`firestore.rules` enforces these guarantees client-side (the Admin SDK used by the backend bypasses them):

- **Authentication required** on all paths — unauthenticated requests are rejected.
- **School scoping** — every read/write checks that `resource.data.school_id` matches the caller's `school_id` custom claim. Users from one school cannot access another school's data even if they know the document IDs.
- **Role enforcement** — mutations to `plates`, `school_admins`, and `plate_scans` (delete) require the `school_admin` role in the custom claim.
- **Client writes to `plate_scans` are blocked** — only the backend service account (Admin SDK) can create scan records. This prevents clients from injecting fake scan events.
- **Catch-all deny** — any path not explicitly listed (including the deleted `imported_data`) returns denied.

## Deploying rules and indexes

```bash
# From the repo root
firebase deploy --only firestore:rules
firebase deploy --only firestore:indexes

# Or both at once
firebase deploy --only firestore
```

Make sure your `firebase.json` includes:

```json
{
  "firestore": {
    "rules": "firestore.rules",
    "indexes": "firestore.indexes.json"
  },
  "hosting": { ... }
}
```
